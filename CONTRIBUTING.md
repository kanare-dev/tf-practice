# コントリビューションガイド

このプロジェクトへの貢献を歓迎します！

## 開発フロー

1. 新しいブランチを作成
2. 変更を実装
3. ADR を作成（重要な決定の場合）
4. テストを実行
5. プルリクエストを作成

## コーディング規約

### Terraform

- `terraform fmt` でフォーマットを統一
- 変数と出力には適切な説明を追加
- リソースには適切なタグを付与

### コミットメッセージ

- 明確で説明的なメッセージを記述
- 変更の理由を含める

## ADR の作成

重要な設計決定をした場合は、ADR を作成してください。

## テスト

変更をプッシュする前に、以下を確認してください：

```bash
# フォーマットチェック
terraform fmt -check -recursive

# バリデーション
cd environments/dev
terraform init
terraform validate

# 実行計画の確認
terraform plan
```

## レビュープロセス

1. プルリクエストを作成
2. CI/CD パイプラインが自動実行
3. レビュー待ち
4. 承認後、マージ
